# 5. Working with commands

* What exactly are commands
    * An executable program, can be compiled binaries and programming written in scripting languages
    * A command built into the shell itself
    * A shell function: miniature shell scripts incorporated into the environment
    * An alias
* It is possible to put more than one command on a line by separating each command with a semicolon

* `type`: display the kind of command the shell will execute
* `which`: display an executable's location
* `help`: get help for shell builtins
* `--help`: display usage information
* `man`: display a program's manual page
* `apropos`: display appropiate commands, search the list of man pages for possible matches based on a search term
* `whatis`: display one-line manual page descriptions
* `info`: display a program's info entry
* `alias name='string'`: alias is a custom shortcout given to a long, complex, or frequently used command
